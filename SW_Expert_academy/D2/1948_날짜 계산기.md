# GPT 피드백
## 로직상 버그 (같은 달인 경우)
=> 개선 버전
```py
T = int(input())
info = [0] * 13
info[1] = 31
info[2] = 28
info[3] = 31
info[4] = 30
info[5] = 31
info[6] = 30
info[7] = 31
info[8] = 31
info[9] = 30
info[10] = 31
info[11] = 30
info[12] = 31

for stage in range(1, T+1):
    m1, d1, m2, d2 = map(int, input().split())
    result = 0

    # 같은 달인 경우
    if m1 == m2:
        result = d2 - d1 + 1

    # 다른 달인 경우
    else:
        # 시작 달: 시작일 ~ 마지막 날
        result += info[m1] - d1 + 1

        # 중간 달들 전부
        for month in range(m1 + 1, m2):
            result += info[month]

        # 끝 달: 1일 ~ d2일까지
        result += d2

    print(f"#{stage} {result}")
```
## 더 깔끔한 방식 (하루 수 누적해서 빼기)
조금 더 수학적으로 쓰면,
각 날짜를 “1월 1일부터 며칠째인지”로 바꾼 뒤에 빼고 +1을 해도 돼.
```py
T = int(input())
days = [0, 31, 28, 31, 30, 31, 30, 31, 31, 30, 31, 30, 31]

def to_day_of_year(m, d):
    return sum(days[1:m]) + d

for stage in range(1, T+1):
    m1, d1, m2, d2 = map(int, input().split())
    start = to_day_of_year(m1, d1)
    end = to_day_of_year(m2, d2)
    result = end - start + 1
    print(f"#{stage} {result}")
```
